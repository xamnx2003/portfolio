Validated Bug Reports

10 real security findings — responsibly disclosed across multiple platforms and programs. Each report includes full reproduction steps, impact analysis, and remediation guidance.

RPT-001 Critical
DOM XSS → ATO
DOM XSS via Import Filename → Full Account Takeover
Account import page renders selected filename using innerHTML instead of textContent, enabling HTML injection inside an authenticated form that already holds the victim's session and a valid CSRF token.
Impact Full account takeover via email change to attacker-controlled mailbox. Also: write-scoped token creation, account deletion.
Basecamp / HackerOne
RPT-002 High
Blind SSRF
Blind SSRF in Export Service via webhookUrl Parameter
The /exportservice/v3/exports/WEBHOOK/accounts endpoint accepts a webhookUrl parameter without validation. Providing a malicious URL causes the server to issue unauthorized HTTP requests to attacker-controlled systems.
Impact Internal network access, port scanning, cloud metadata exfiltration, potential chained RCE via internal service exploitation.
REDACTED SaaS Platform
RPT-003 Critical
Path Traversal → RCE
libcurl SFTP QUOTE Path Traversal → Arbitrary File Write & RCE
Curl_get_pathname in lib/vssh/curl_path.c fails to sanitize ../ sequences in user-provided SFTP QUOTE paths. An attacker with SFTP access can rename, delete, or write files outside their home directory.
Impact Full server compromise via authorized_keys overwrite or system script replacement. Breaks SFTP directory restriction model entirely.
libcurl / curl 8.15.0
RPT-004 High
Mutation XSS
Mutation-Based Stored XSS in Trix Editor v2.1.8
Trix Editor 2.1.8 is vulnerable to mXSS via MathML mutation — a DOMPurify bypass vector. A crafted data-trix-attachment attribute containing HTML5 content type triggers JavaScript execution upon paste.
Impact Arbitrary JavaScript execution in victim session context. Unauthorized actions, session hijacking, sensitive data disclosure.
Trix Editor / Open Source
RPT-005 High
IDOR
IDOR Leads to Unauthorized User Profile Modification
POST /app/updateUser performs no authorization check on the target user ID. An attacker can substitute any victim's ID and email in the JSON body to overwrite their profile data.
Impact Attacker can silently modify username, address, mobile number, company name, and company size of any account.
target.com.ng / HackerOne
RPT-006 Critical
SQL Injection
SQL Injection in DoD .mil Application — 24 Databases Exposed
The selMajcom parameter in RequestAccess.asp is unsanitized and injectable. Automated via sqlmap, 24 internal databases were enumerated without authentication on a US Department of Defense asset.
Impact Full database enumeration of 24 internal DoD databases including master, NAFRIS, W2DATA, ORCA. Unauthenticated access.
US DoD (.mil) / Bug Bounty
RPT-007 Critical
Auth Bypass → ATO
Missing Auth Check Enables Password Reset on Any Account
The one-time password change flow in Login.aspx (.mil) applies no authentication check on the txtEMail parameter. Any attacker can submit a password change for any email address, resulting in full account takeover.
Impact Full account takeover on any user of the .mil platform. No authentication required — attacker only needs a victim email address.
US DoD (.mil) / Bug Bounty
RPT-008 High
XXE
XXE in Elastic Enterprise Search Web Crawler via Sitemap
The App Search web crawler parses sitemap.xml files without disabling external entity resolution. A malicious robots.txt pointing to an attacker-controlled sitemap triggers OOB XXE data exfiltration.
Impact Hostname confirmed exfiltrated. Potential for credentials, AWS metadata, and arbitrary single-line file reads from the crawler host.
Elastic Enterprise Search
RPT-009 High
Blind SQLi
Blind SQL Injection in inDrive Prize Endpoint
The number_trips path segment in /api/ten-drives/custom-winners/ is passed unsanitized to a SQL query. Boolean-based injection (OR 1=1 vs OR 1=2) produces differential responses, confirming exploitability.
Impact PostgreSQL 14.8 version confirmed. Full database enumeration, data tampering, and potential authentication bypass possible.
inDrive / Bug Bounty
RPT-010 Critical
ORM SQLi
SQL Injection via Django Q Object _connector Key
WhereNode.as_sql in Django's ORM uses unsafe string formatting for the connector value. When Q(**user_dict) pattern is used, an attacker-controlled _connector key injects raw SQL into the WHERE clause bypassing all ORM parameterization.
Impact Access control bypass (OR 1=1), full table exfiltration of any model, DoS via complex queries. Admin users confirmed retrievable.
Django ORM / Open Source
